TROUBLESHOOTING
───────────────────────────────────────────────────────────────────────────────

PROBLEM: Checks tab doesn't appear
SOLUTION:
  1. Hard refresh: Ctrl+Shift+R (or Cmd+Shift+R on Mac)
  2. Clear browser cache: Chrome → Settings → Privacy → Clear data
  3. Check console for errors: F12 → Console tab
  4. Verify file exists: Frontend/app/(dashboard)/credits/page.tsx

PROBLEM: Date filter shows but no table
SOLUTION:
  1. Check DevTools Network tab: F12 → Network
  2. Click Submit on filter
  3. Look for "my-payments" request
  4. If request fails → Backend API issue
  5. If request succeeds but shows empty → No data in database

PROBLEM: Table is empty or shows old data
SOLUTION:
  1. Verify database migration completed
  2. Check if pharmacy has check records
  3. Try selecting different date range
  4. Open DevTools Console for API error messages

PROBLEM: PDF doesn't open when clicking check number
SOLUTION:
  1. Verify Puppeteer installed: npm list puppeteer
  2. Check backend logs for PDF generation errors
  3. Try opening browser console: F12 → Console
  4. Verify check number is clickable (should be blue link)

PROBLEM: Page works but looks wrong (styling)
SOLUTION:
  1. Verify TailwindCSS is built: npm run build
  2. Check browser DevTools → Styles tab
  3. Look for missing CSS classes
  4. Hard refresh and clear cache
  5. Restart dev server: npm run dev

PROBLEM: TypeScript errors in IDE
SOLUTION:
  1. Run: npm run build
  2. Fix any errors shown
  3. Restart IDE TypeScript server: Ctrl+Shift+P → "Restart TS Server"

PROBLEM: Mobile view looks broken
SOLUTION:
  1. Open DevTools: F12
  2. Click responsive design: Ctrl+Shift+M
  3. Set viewport: 390x844 (iPhone)
  4. If table breaks → May need horizontal scroll
  5. Check if buttons are clickable
