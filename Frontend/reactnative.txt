### Route Categories
 
1. **Public Routes** (4): Login, Register, Forgot Password, Reset Password
2. **Protected Routes** (36): All dashboard routes require authentication
3. **Dynamic Routes** (6): Routes with dynamic `[id]` parameters
4. **API Routes** (6): Backend API endpoints
 
### Route Protection
 
All dashboard routes are protected by middleware (`middleware.ts`) and require:
- Valid authentication token (`auth_token` cookie)
- Redirect to `/login` if not authenticated
- Preserve intended destination for post-login redirect
 
### Special Routes
 
- **Root Route (`/`)**: Automatically redirects to `/login`
- **External Redirect Routes**:
  - `/marketplace/checkout/success` - Handles Stripe redirects
  - `/subscription` - Handles subscription redirects
- **Protected Routes**: All routes under `/dashboard`, `/products`, `/marketplace`, etc.
