## 🧪 Frontend Testing Flow

### **Test 1: Navigate to Credits Page**

Steps:
1. Go to your Pharmacy Portal
2. Click on "Credits" in the sidebar menu
3. You should see the /credits page load

Expected Result:
- Page displays two tabs at the top: "Credits" and "Checks"
- Credits tab is active by default
- Existing credits data is displayed in a table

---

### **Test 2: Switch to Checks Tab**

Steps:
1. On the Credits page, click the "Checks" tab button
2. Observe the page transition

Expected Result:
- Tab switches to "Checks" without page reload
- New content appears with a date range filter at the top
- A table appears below with check history
- You should see columns: Return Date, Ref #, Date Paid, Check #, Amount, etc.

---

### **Test 3: Test Date Range Filter - Presets**

Steps:
1. Click the date range dropdown (currently shows "All Dates")
2. Select different preset options:
   - "This Month"
   - "Last Month"
   - "This Quarter"
   - "Last Year"
3. Click "Submit" button after each selection
4. Observe table updates

Expected Result:
- Table filters and displays only checks within selected date range
- Page shows loading spinner during fetch
- Results update within 1-2 seconds
- Record count changes based on date range

---

### **Test 4: Test Custom Date Range**

Steps:
1. Click date range dropdown
2. Select "Custom" option
3. Two date input fields should appear
4. Enter or pick a date range (e.g., Jan 1, 2026 to Mar 31, 2026)
5. Click "Submit" button

Expected Result:
- Loading spinner appears
- Table filters to show only checks within that date range
- If no checks exist: "No checks found" message appears
- Validation error if "To Date" is before "From Date"

---

### **Test 5: Test Tooltips (OCS vs POR)**

Steps:
1. In the Checks tab, hover over the OCS and POR info icons (i)
2. Read the tooltip text

Expected Result OCS Tooltip:
"You select the time-frame in which you want your credit due from RSI to be issued in a single check..."

Expected Result POR Tooltip:
"Certain characteristics of a return may cause drug manufacturers to require proof..."

---

### **Test 6: Test Check Number Clickable Links**

Steps:
1. In the Checks table, locate the "Check #" column
2. Look for a check number like "216461"
3. The check number should be styled as a link (blue/underlined)
4. Click on a check number

Expected Result:
- PDF opens in a new browser tab
- Check PDF displays with complete check information:
  - RSI company header
  - "PAY TO THE ORDER OF:" section
  - Check amount ($X,XXX.XX)
  - Amount in words (e.g., "TWO THOUSAND EIGHT HUNDRED...")
  - Manufacturer breakdown sections
  - Credit summary section

---

### **Test 7: Test Table Sorting**

Steps:
1. In the Checks table, click column headers:
   - "Return Date" header
   - "Check Amount" header
   - "Date Paid" header
2. Click again to reverse sort order

Expected Result:
- Table rows reorganize based on clicked column
- Clicking again reverses order (ascending → descending)
- Sorting completes within 500ms

---

### **Test 8: Test Table Search**

Steps:
1. Find the search box in the Checks table (usually at top)
2. Type a check number: "216461"
3. Table should filter in real-time
4. Try searching for:
   - A return reference: "3S15L"
   - A partial date: "02/2026"

Expected Result:
- Results filter immediately as you type (< 300ms)
- Only matching rows display
- Clear button appears when search has text

---

### **Test 9: Test Pagination**

Steps:
1. Scroll to bottom of Checks table
2. You should see pagination controls
3. Click "[Next >]" button
4. Click "[< Previous]" to go back

Expected Result:
- Page indicator updates (Page 1 → Page 2, etc)
- Table rows update with new records
- Previous button disabled on page 1
- Next button disabled on last page

### **Test 10: Test Totals Footer**

Steps:
1. Look at the bottom of the Checks table
2. Find the "TOTALS" row with calculations

### **Test 11: Test Responsive Design (Mobile)**

4. Navigate through Checks tab features

### **Test 12: Test Loading & Error States**

Steps:
1. Apply a date filter and submit
2. Observe loading spinner while data fetches

## 🔧 Troubleshooting

If Checks tab doesn't appear:
- Hard refresh page (Ctrl+Shift+R)
- Clear browser cache
- Check console for errors (F12 → Console)

If table is empty:
- Backend database migration not run yet
- Check API response in DevTools → Network tab

If PDF doesn't open:
- Puppeteer not installed
- Backend not running
- Check backend logs
